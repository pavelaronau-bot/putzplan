# ADR-0007. Стек Sprint 1: FastAPI + Python 3.12

**Дата:** 20.07.2026 · **Статус:** принято · **Замещает:** ADR-0005

## Контекст
Каркас Gate 1 был написан на Node.js, чтобы быстро проверить схему и права
на живой базе. Техническое задание Sprint 1 утвердило другой стек:
FastAPI, SQLAlchemy 2 async, Alembic, pytest.

## Решение
Backend переписан на Python 3.12 + FastAPI. Сохранены без изменений:
схема базы, роли БД, `SECURITY DEFINER`-функции аутентификации, хеш-цепочка
журнала, SQL-тесты укрепления — они не зависят от языка приложения.

Ключевые соответствия:

| Было (Node.js, Gate 1) | Стало (Python, Sprint 1) |
|---|---|
| `pg` + два пула | SQLAlchemy async + два движка |
| `@node-rs/argon2` | `argon2-cffi`, те же параметры (19 МиБ, 2, 1) |
| Ручная маршрутизация на `node:http` | FastAPI с `Depends` и генерацией OpenAPI |
| Проверка прав в обработчике | Централизованная зависимость `require(permission)` |
| Тесты на Node | pytest: 26 unit + 24 integration |

## Последствия
- Контракт OpenAPI 3.1 генерируется из кода, а не пишется руками.
- Валидация запросов и ответов — на Pydantic, единая схема ошибки.
- Каркас на Node.js больше не поддерживается; артефакты Gate 1 по базе
  остаются в `infrastructure/db` и используются как есть.
- Alembic-ревизия `0001` применяет те же проверенные SQL-файлы, поэтому
  результат миграции идентичен прошедшему Gate 1.
