# ADR-0004. DEFAULT-партиции создаются

**Дата:** 20.07.2026 · **Статус:** принято

## Контекст
Если плановое задание создания партиций не отработает, вставка в журнал
завершится ошибкой. Потеря записи журнала недопустима; отказ бизнес-операции
из-за отсутствия партиции — тоже.

## Решение
У каждой партиционированной таблицы есть DEFAULT-партиция. Непустая DEFAULT
считается аварией уровня P2 и отслеживается функцией `partition_default_rows()`.
Для разбора предусмотрена процедура `redistribute_default_partition()`.

## Альтернатива, которую отклонили
Отказ от DEFAULT с расчётом только на мониторинг: при пропуске задания
приложение теряло бы записи журнала — цена ошибки выше выигрыша.

## Последствия
- Вставка не падает при пропуске задания.
- Присоединение новой партиции при непустой DEFAULT требует перераспределения
  строк — процедура описана и выполняется в окне низкой нагрузки.
- Горизонт 18 месяцев делает срабатывание маловероятным: запас 561 день.
