# ADR-0006. bigint отдаётся числом в JSON

**Дата:** 20.07.2026 · **Статус:** принято

## Контекст
Драйвер `pg` по умолчанию возвращает `bigint` строкой, чтобы не потерять
точность. В API это привело к дефекту: клиент сравнивал `chain_seq` как
строки, и `'10'` оказывалось меньше `'9'`. Нашёл автоматический тест API.

## Решение
Зарегистрирован разбор типов 20 (`int8`) и 1700 (`numeric`) в число.
Все наши счётчики заведомо меньше 2^53.

## Последствия
- Порядок и сравнение в клиентах корректны.
- При появлении значений свыше 2^53 (не ожидается) потребуется вернуть строки
  и договориться о формате с клиентами.
- Денежные значения `numeric` приходят числами: для расчётов на клиенте
  использовать только отображение, вычисления остаются на сервере.
