# ADR-0003. Узкие SECURITY DEFINER-функции вместо BYPASSRLS для входа

**Дата:** 20.07.2026 · **Статус:** принято

## Контекст
Аутентификация происходит до того, как известен арендатор, поэтому RLS
не может отфильтровать пользователя: запрос к `users` возвращает 0 строк
и вход невозможен. Очевидное решение — выдать рабочей роли `BYPASSRLS` —
открыло бы ей чтение всех данных всех компаний.

## Решение
Рабочая роль остаётся `NOBYPASSRLS`. Открыт узкий набор `SECURITY DEFINER`-функций:
`auth_find_user`, `auth_session_lookup`, `auth_user_permissions`,
`auth_register_failure`, `auth_register_success`, `auth_revoke_user_sessions`,
`auth_record_attempt`. Каждая возвращает ровно те поля, которые нужны для входа.

## Последствия
- Компрометация рабочей роли не даёт доступа к данным других компаний.
- Функции становятся частью контракта безопасности: их изменение требует ревью.
- Создание арендатора выполняется отдельной ролью провижининга — рабочая роль
  не может вставлять строки в `companies`, что подтверждено тестом.
