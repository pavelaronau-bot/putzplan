## 1. Критический дефект параллельной ротации — закрыт

### Воспроизведение до исправления

```
победителей: 1 | коды: [200, 409 × 9]
токен победителя после гонки: 401 session_revoked
```

Проигравший запрос дожидался блокировки, видел старую сессию отозванной,
трактовал это как кражу и отзывал всё семейство — вместе с сессией победителя.

### Решение

Миграция `0004` вводит явное состояние замены: колонки `rotated_at`
и `replaced_by_session_id`, grace-window 30 секунд.

| Состояние старой сессии | Результат | Действие |
|---|---|---|
| активна | `rotated` | создаётся преемник |
| заменена, в пределах окна | `race` | **ничего не отзывается**, событие `REFRESH_RACE_DETECTED` |
| заменена, вне окна | `reuse` | отзыв семейства, событие `REFRESH_REUSE_DETECTED` |
| отозвана по другой причине | `reuse` | отзыв семейства |
| истекла / пользователь неактивен / не найдена | `expired` / `inactive_user` / `not_found` | 401 / 403 / 401 |

Состояние `race_lost` удалено как недостижимое под блокировкой строки;
отсутствие проверяется тестом. Raw refresh-токен в базе не хранится —
только SHA-256 хеш.

## 2. Миграция 0004: настоящий downgrade

Прежняя версия `downgrade()` создавала заглушку, всегда возвращавшую
`not_found` — после отката приложение переставало обновлять токены.

Исправлено: `downgrade()` восстанавливает **полную реализацию из ревизии 0003**
(состояния `rotated`, `reuse`, `race_lost`, `expired`, `inactive_user`,
`not_found`, вставка новой сессии) и дополнительно создаёт совместимую обёртку
с шестым аргументом `p_grace_seconds`, который игнорируется. Обёртка нужна,
чтобы код текущего релиза, вызывающий функцию с шестью аргументами, продолжал
работать сразу после отката: семантика при этом возвращается к прежней.

### Тест цикла миграций

`tests/test_migration_cycle.py` выполняет реальные HTTP-сценарии на каждом шаге:

| Этап | Ревизия | Вход | Обновление токена |
|---|---|---|---|
| upgrade head | 0004 | ✅ 200 | ✅ 200 |
| downgrade -1 | 0003 | ✅ 200 | ✅ 200 |
| upgrade head | 0004 | ✅ 200 | ✅ 200 |

Второй тест проверяет, что после отката в теле функции присутствуют все
состояния и вставка сессии, а колонки ревизии 0004 удалены — то есть
восстановлена реализация, а не заглушка.
