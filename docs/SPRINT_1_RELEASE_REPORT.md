| Проверка | Инструмент | Результат |
|---|---|---|

| SAST | bandit `-ll` | 0 высоких, 0 средних после разбора; 4 находки B608 признаны ложными и снабжены пояснением |
| Секреты в коде | grep-правило в CI | не найдены |
| Инъекции SQL | собственный тест с четырьмя нагрузками | таблицы целы, данные не утекли |
| Пароли | Argon2id (19 МиБ, 2 итерации) | проверено тестами |
| Журнал без секретов | тест на подстроки | пароли и токены отсутствуют |

Найденные и исправленные в ходе спринта дефекты:

1. **Отзыв сессий откатывался.** При обнаружении повторного использования
   refresh-токена исключение бросалось внутри транзакции, и отзыв всей цепочки
   откатывался вместе с ней — украденный токен оставался рабочим. Исправлено:
   отзыв фиксируется до выброса ошибки. Найдено интеграционным тестом.
2. **Повреждённый хеш ронял вход** (`UnicodeEncodeError` вместо отказа).
3. **Двойное использование параметра** в `INSERT` ломало создание пользователя
   (asyncpg не выводил тип) — добавлено явное приведение.
4. **Сид не обновлял e-mail** при конфликте по `id`.
5. **Кнопка входа блокировалась молча** при неверном вводе — пользователь не
   видел причины. Поведение исправлено, проверка выполняется при отправке.

## 9. Известные ограничения

1. **Playwright не запускался** в этой среде — браузеры недоступны для загрузки.
   Сценарий написан и включён в CI, но локального подтверждения нет.
2. **Docker Compose не поднимался** здесь: демон Docker в среде отсутствует.
   Файлы написаны, порядок зависимостей и healthcheck заданы, но фактический
   запуск `docker compose up --build` предстоит проверить на машине с Docker.
   Все компоненты проверены запуском напрямую.
3. **Ограничение частоты входов хранится в памяти процесса.** При нескольких
   репликах счётчик не общий. Redis поднят в Compose как инфраструктурная
   готовность; вынос счётчика — задача Sprint 2.
4. **Проверка прав кэшируется на 60 секунд.** Отзыв права действует
   с задержкой до минуты; отзыв сессии — немедленно.
5. **mypy запускается в CI без блокировки** (`|| true`): типизация доводится
   постепенно, ruff при этом блокирует сборку.
6. **Реализован один вертикальный срез.** Планирование, объекты, клиенты,
   мобильный кабинет, табель, финансы — этапы 1–5 дорожной карты.
